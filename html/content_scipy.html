SciPy

SciPy is an open source library of scientific tools for Python.

The VFGEN SciPy Command

Python files that can be used with the SciPy ODEINT routine are created by the command

$ vfgen scipy vector_field_file.vf

The files created are [name].py and, if the option demo=yes option is given, [name]_demo.py, where [name] is the name of the vector field given in the vector field file. The Python file [name].py will contain the functions vectorfield and jacobian.

Options

func If the option func=yes is given, VFGEN also converts any user-defined functions in the vector field file into functions in the Python file. The names of the functions will be the same as those given in the vector field file.
Default: func=no
demo If the option demo=yes is given, the file [name]_demo.py will contain a Python script for a command-line ODE solver for the vector field. The example below shows how this file can be used.
Default: demo=no
tfirst The system functions passed to the solver scipy.integrate.odeint() expect the first parameter to be the state vector and the second to be the independent variable (typically the time t). The other ODE solvers in scipy.integrate use the opposite convention: t is first, then the state vector. The code generated by VFGEN follows the default convention of odeint. If, however, tfirst is "yes", the code will be written to use the opposite convention. Then the generated functions may be used with any of the solvers from scipy.integrate. To use them with odeint(), include the argument `tfirst=True`. The options applies to *all* the generated functions: the main system definition, the Jacobian, and any additional user-defined functions.
Default: tfirst=no

$ vfgen scipy:func=yes,demo=yes pendulum.vf

are pendulum.py and pendulum_demo.py. Run the demonstration program with a command such as
$ python pendulum_demo.py theta=3.1 b=0.5 stoptime=10 > pend.dat
